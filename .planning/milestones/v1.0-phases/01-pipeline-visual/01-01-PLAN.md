  <read_first>

  <action>
Create `scripts/pdf-to-png.py` — a Python script using PyMuPDF (fitz) to convert regional PDFs to PNGs.

Requirements:
- Import `fitz` (PyMuPDF), `os`, `sys`, `argparse`, `json`
- Accept CLI args: `--region=SLUG` (single region), `--all` (all 16), `--dpi=300` (default 300), `--output-dir=DIR` (default `data/snapshots`)
- Find PDFs in `data/snapshots/` matching pattern `mineduc-{region}.pdf`
- For each PDF, render ALL pages to PNG at specified DPI using `page.get_pixmap(matrix=fitz.Matrix(dpi/72, dpi/72))`
- Output naming: `{region}-p{pagenum}.png` (1-indexed) in the output directory
- Also produce a manifest JSON written to stdout: `{"region": "aysen", "pdf": "mineduc-aysen.pdf", "pages": 5, "pngs": ["aysen-p1.png", ...]}`
- If `--all`, process these 16 region slugs: arica-y-parinacota, tarapaca, antofagasta, atacama, coquimbo, valparaiso, metropolitana, ohiggins, maule, nuble, biobio, araucania, los-rios, los-lagos, aysen, magallanes
- If PDF not found for a region, print warning to stderr and continue
- Exit code 0 if at least one region processed, 1 if none

The 16 PDFs already exist in `data/snapshots/` with names like `mineduc-aysen.pdf`, `mineduc-metropolitana.pdf`, etc.
  </action>
  <verify>
    <automated>python scripts/pdf-to-png.py --region=aysen --output-dir=data/snapshots 2>&amp;1 | head -20</automated>
  </verify>
  <acceptance_criteria>

    - scripts/pdf-to-png.py contains the string `mineduc-` for PDF filename pattern
    - Running `python scripts/pdf-to-png.py --region=aysen --output-dir=data/snapshots` produces at least one `aysen-p*.png` file in data/snapshots/
    - Script outputs JSON manifest to stdout with keys "region", "pdf", "pages", "pngs"
  </acceptance_criteria>
  <done>Script converts any regional PDF to PNGs at 300 DPI with correct naming. Tested on aysen.</done>

  <action>
Create `scripts/organize-snapshots.js` — a Node.js script (CommonJS, 'use strict', var) that:

1. Reads `data/extraction-tests/TODAS-REGIONES-visual-extraction.json`
2. For each region in `.regions`, reads the `images` array (e.g., `["aysen-p2.png", "aysen-p3.png"]`)
3. Copies those PNGs from `data/extraction-tests/` to `data/snapshots/` with canonical naming: `{regionSlug}-tabla-p{N}.png` where N is sequential (1, 2, 3...) regardless of original page number
4. Also creates a manifest file `data/snapshots/png-manifest.json` with structure:
```json
{
  "generated": "2026-03-24T...",
  "regions": {
    "aysen": {
      "table_pngs": ["aysen-tabla-p1.png", "aysen-tabla-p2.png"],
      "source_pages": [2, 3],
      "grupo": "SUR"
    }
  }
}
```
5. CLI: `node scripts/organize-snapshots.js` (no args needed)
6. Prints summary: `Organized X PNGs for Y regions`
7. Does NOT delete source files in extraction-tests/

Use `var fs = require('fs');` and `var path = require('path');` only. No npm dependencies.
  </action>

  </verify>
