  <action>
Create `scripts/extract-visual.js` — CommonJS, 'use strict', var — that extracts structured calendar data from PNG images via a multimodal LLM API.

**Architecture:**
1. Read `data/snapshots/png-manifest.json` to get list of table PNGs per region
2. For each region, read the table PNG(s) as base64
3. Call multimodal LLM API (Anthropic Claude or OpenAI — configurable via env var `EXTRACTION_API`) with a structured prompt
4. Parse LLM response into the gold standard JSON structure
5. Write results to `data/visual-extraction.json`

**CLI interface:**
- `node scripts/extract-visual.js` — process all 16 regions
- `node scripts/extract-visual.js --region=aysen` — single region
- `node scripts/extract-visual.js --local` — use existing extraction (data/extraction-tests/TODAS-REGIONES-visual-extraction.json) instead of calling API, for offline/testing
- `node scripts/extract-visual.js --dry-run` — show what would be processed without API calls

**Environment variables:**
- `EXTRACTION_API` — "anthropic" (default) or "openai"
- `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` — API key for the chosen provider
- `EXTRACTION_MODEL` — model name override (default: "claude-sonnet-4-20250514" for anthropic, "gpt-4o" for openai)

**Prompt engineering:**
The prompt sent with each PNG must instruct the LLM to:
- Identify whether the table is semestral, trimestral, or both
- Extract every row: label (milestone name), date(s) in ISO format (YYYY-MM-DD), raw_text (exactly as written in table), day_of_week if visible
- For date ranges (receso, fiestas patrias), use date_start and date_end
- Return valid JSON matching the gold standard structure
- Include any notes about anomalies (dates that seem wrong, missing cells)

**LLM API call implementation:**
Use `https` module (no npm deps). For Anthropic:
```
POST https://api.anthropic.com/v1/messages
Headers: x-api-key, anthropic-version: 2023-06-01, content-type: application/json
Body: { model, max_tokens: 4096, messages: [{ role: "user", content: [{ type: "image", source: { type: "base64", media_type: "image/png", data: BASE64 }}, { type: "text", text: PROMPT }]}]}
```

**--local mode (critical for testing):**
When `--local` flag is used, skip API calls entirely. Instead:
1. Read `data/extraction-tests/TODAS-REGIONES-visual-extraction.json`
2. For each region, expand the 6-field summary into the full gold standard structure using the known mapping:
   - inicio -> "Ingreso de estudiantes NT1 a 4° Medio" milestone
   - vacacionesInicio/vacacionesFin -> "Receso de invierno" milestone with date_start/date_end
   - fiestasPatriasInicio/fiestasPatriasFin -> "Fiestas Patrias" milestone
   - finAno -> "Último día de clases establecimientos con JEC" milestone
3. Write the same output format as API mode

**Output format** (`data/visual-extraction.json`):
```json
{
  "_meta": { "pipeline": "visual", "date": "...", "api": "anthropic|openai|local" },
  "regions": {
    "aysen": { "_meta": {...}, "year": 2026, "semestral": [...], "trimestral": [...] },
    ...
  }
}
```

**Error handling:**
- API timeout: 90 seconds per region, retry once
- Rate limiting: 3-second delay between API calls
- Parse failure: log error, write `{"error": "..."}` for that region, continue
- At end, print summary: `Extracted N/16 regions (M errors)`

**Reuse from extract-from-pdf.js:**
- Copy `REGION_MAP` object (all slug mappings)
- Copy `MONTHS_ES` array
- Copy CLI parsing pattern (process.argv.slice(2), indexOf)
- Same 'use strict' + var pattern
