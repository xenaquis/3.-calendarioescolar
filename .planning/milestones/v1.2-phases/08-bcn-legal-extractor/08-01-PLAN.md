    Create a single Python script with these sequential phases. Use `#!/usr/bin/env python3` shebang. All stdlib imports except `anthropic` SDK.

    **Constants and corrections map:**
    ```python
    BCN_API_BASE = 'https://nuevo.leychile.cl/servicios'
    ```

    **Function: get_idnorma(afirmaciones, source_id)**
    - If source_id in IDNORMA_CORRECTIONS, return the corrected value
    - Otherwise, extract idNorma from `afirmaciones['sources'][source_id]['api_endpoint']` using `re.search(r'idNorma=(\d+)', endpoint)`
    - Raise ValueError if no idNorma found

    **Function: fetch_norma_json(id_norma, timeout=30)**
    - Build URL: `'{}/Navegar/get_norma_json?idNorma={}'.format(BCN_API_BASE, id_norma)`
    - Use `urllib.request.Request` with `User-Agent: Mozilla/5.0` header
    - Return `json.loads(resp.read().decode('utf-8'))`
    - Catch `urllib.error.HTTPError` and `urllib.error.URLError`, re-raise as RuntimeError with descriptive message

    **Function: extract_articles(norma_data)**
    - Read `estructura` array and `html` dict (keyed by item `i`)
    - For each part in `estructura` where `'art' in name.lower()` and `item_id in html_items`:
      - Call `extract_text()` on the HTML string
      - Append `{'numero': name, 'item_id': item_id, 'texto': texto}` to result list
    - Return list of article dicts

    **Function: extract_text(html_string)**
    - `re.sub(r'<[^>]+>', ' ', html_string)` to strip tags
    - `html_module.unescape(text)` to decode entities
    - `re.sub(r'\s+', ' ', text).strip()` to normalize whitespace
    - Return cleaned text

    **Function: compute_hash(text)**
    - Return `hashlib.sha256(text.encode('utf-8')).hexdigest()`

    **Function: get_feriado_claims(afirmaciones)**
    - Iterate `afirmaciones['claims']` array
    - Filter: `claim['id'].startswith('feriado')` AND `claim.get('source_id', '').startswith('bcn-')`
    - Group by `source_id` into a dict of lists
    - Return dict (4 keys: bcn-ley-2977, bcn-ley-19668, bcn-ley-20148, bcn-ley-21357)

    **Function: build_claude_client()**
    - Read `ANTHROPIC_API_KEY` from `os.environ.get('ANTHROPIC_API_KEY')`
    - If missing: print to stderr `'ERROR: ANTHROPIC_API_KEY not set.\n  Run: export ANTHROPIC_API_KEY=sk-ant-...'` and `sys.exit(1)`
    - Return `anthropic.Anthropic(api_key=api_key)`

    **Function: identify_articles(client, ley_id, articles, claims)**
    - Build articles_block: `'\n'.join('{}: {}'.format(a['numero'], a['texto']) for a in articles)`
    - Build claims_block: `'\n'.join('- {} | "{}" | hint: {}'.format(c['data_key'], c['claim'], c.get('source_reference', '')) for c in claims)`
    - Prompt template (Spanish, JSON-only response):
      ```
      Ley: {ley_id}

      Articulos de la ley:
      {articles_block}

      Claims que debes mapear a articulos:
      {claims_block}

      Para cada claim, identifica el articulo que lo respalda.
      Responde SOLO con JSON valido, sin texto adicional:
      {{"data_key_1": {{"articulo_numero": "PRIMERO", "inciso": "2"}}, ...}}
      Si no puedes identificar el articulo para un claim, usa null para ambos campos.
      ```
    - Call `client.messages.create(model='claude-haiku-4-5', max_tokens=1024, messages=[...])`
    - Parse response: try `json.loads(response_text)`, fallback to `re.search(r'\{.*\}', response_text, re.DOTALL)` regex extraction
    - Return dict mapping data_key to {articulo_numero, inciso}

    **Function: update_entry(existing, new_texto, new_hash, ley_id, articulo_numero, inciso)**
    - If existing is None, create new entry dict with all fields set to None initially
    - If existing has a hash and it differs from new_hash: set `texto_anterior` to existing `texto_verbatim`
    - Update: texto_verbatim, hash_sha256, last_checked (UTC ISO format `datetime.utcnow().strftime('%Y-%m-%dT%H:%M:%SZ')`), ley_id, articulo_numero, inciso
    - Return the entry dict

    **Function: main()**
    - Parse args with argparse: `--dry-run` (fetch and show articles but no writes, no Claude API), `--force` (re-identify even if articulo_numero exists)
    - Resolve paths relative to script location: `script_dir = os.path.dirname(os.path.abspath(__file__))`, `data_dir = os.path.join(script_dir, '..', 'data')`
    - Load `afirmaciones.json` from `data_dir`
    - Load existing `legal-articles.json` if it exists (for idempotent updates)
    - Group feriado claims by law using `get_feriado_claims()`
    - Print summary: `'Found {} BCN-sourced feriado claims across {} laws'.format(total, len(by_law))`
    - For each law (sorted by source_id for deterministic order):
      1. Get idNorma via `get_idnorma()`
      2. Fetch norma JSON via `fetch_norma_json()`
      3. Extract articles via `extract_articles()`
      4. Print: `'  {} — {} articles extracted'.format(source_id, len(articles))`
      5. Add 1-second delay between law fetches (`time.sleep(1)`) to avoid rate limiting
      6. If not dry-run AND (any claim for this law needs identification OR --force):
         - Build Claude client (lazy, only when needed)
         - Filter claims needing identification: those where existing entry has `articulo_numero` is None/empty, OR --force flag
         - Call `identify_articles()` with only claims needing identification
         - Print identification results
      7. For each claim in this law:
         - Get the full article text for the identified article (match by articulo_numero from identification, or from existing entry)
         - Compute hash of article text
         - Call `update_entry()`
         - Store in output dict keyed by `data_key`
    - If unidentified claims remain: mark them with `"status": "unidentified"` and print warning
    - Build `_meta`: `{"generated_at": now_iso, "script_version": SCRIPT_VERSION, "total_claims": len(output_minus_meta)}`
    - Write `legal-articles.json` with `json.dump(output, f, ensure_ascii=False, indent=2)`
    - Print summary of results

    **Error handling:**
    - BCN fetch failure: catch and print descriptive error, `sys.exit(1)`
    - Missing ANTHROPIC_API_KEY: print instructions, `sys.exit(1)`
    - Claude API error: catch `anthropic.APIError`, print error, continue to next law (partial results OK)
    - Claude JSON parse failure: print warning, mark claims as unidentified, continue
