  <action>
Create `scripts/merge-claims.js` that:

1. Reads `data/afirmaciones.json` and `data/legal-articles.json`
2. Produces `data/claims.json` with this exact structure:

```json
{
  "_meta": {
    "version": "2.0.0",
    "generatedAt": "<ISO timestamp>",
    "description": "Modelo unificado de claims. Fuente de verdad para todas las afirmaciones del sitio.",
    "totalClaims": <number>,
    "model": "claim-centric-unified",
    "merged_from": ["afirmaciones.json", "legal-articles.json"]
  },
  "sources": { <copied from afirmaciones.json sources> },
  "claims": [
    {
      "id": "<from afirmaciones>",
      "tags": ["<from afirmaciones>"],
      "pregunta": "<derived: what question does this claim answer>",
      "respuesta": "<from afirmaciones.claim field>",
      "displayed_value": "<from afirmaciones>",
      "data_key": "<from afirmaciones>",
      "data_path": "<from afirmaciones>",
      "source_id": "<from afirmaciones>",
      "source_reference": "<from afirmaciones>",
      "fuente_url": "<resolved from sources[source_id].url, or null>",
      "fuente_tipo": "<from sources[source_id].type: 'pdf'|'xml'|'html', or null>",
      "extracto_verbatim": "<from legal-articles[data_key].texto_verbatim if exists, else null>",
      "hash_sha256": "<from legal-articles[data_key].hash_sha256 if exists, else null>",
      "last_checked": "<from legal-articles[data_key].last_checked if exists, else null>",
      "articulo_numero": "<from legal-articles[data_key].articulo_numero if exists, else null>",
      "inciso": "<from legal-articles[data_key].inciso if exists, else null>",
      "texto_anterior": "<from legal-articles[data_key].texto_anterior if exists, else null>",
      "verification_method": "<from afirmaciones>",
      "verification_tier": "<from afirmaciones>",
      "expected_check": "<from afirmaciones if exists, else null>",
      "last_verified": "<from afirmaciones>",
      "last_verified_by": "<from afirmaciones>",
      "status": "<from afirmaciones>"
    }
  ]
}
```

3. The `pregunta` field is derived from the claim text by converting it to a question. Logic:
   - If claim starts with date/event info, derive "Cuando es [event]?" or "Cuantos [item] hay?"
   - Simple heuristic: store a mapping object at top of script for the 50 known claims. Example mappings:
     - "fecha-inicio-clases" -> "Cuando comienzan las clases 2026?"
     - "feriado-ano-nuevo" -> "Cuando es el feriado de Ano Nuevo 2026?"
     - "total-regiones" -> "Cuantas regiones tienen calendario escolar?"
     - "region-inicio" -> "Cuando es el ingreso de estudiantes por region?"
   - For any claim without explicit mapping, use the claim text as-is (fallback)

4. The `respuesta` field copies the `claim` text from afirmaciones.json (this IS the human-readable answer)

5. Join logic for legal-articles enrichment:
   - For each afirmaciones claim, check if `claim.data_key` exists as a key in legal-articles (excluding `_meta`)
   - If match found: copy `texto_verbatim`, `hash_sha256`, `last_checked`, `articulo_numero`, `inciso`, `texto_anterior`
   - If no match: set all those fields to `null`

6. The `fuente_url` field resolves from sources: `afirmaciones.sources[claim.source_id].url` or null if source_id is null

7. Print summary: total claims, enriched count (with verbatim), unenriched count

Use project conventions: `var`, `require('fs')`, `require('path')`, no ES modules, no npm dependencies.

Add npm script: The executor should NOT modify package.json — the merge script is run manually via `node scripts/merge-claims.js`.
