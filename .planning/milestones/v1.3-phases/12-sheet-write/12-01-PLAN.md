**Header comment** (same style as sync-from-sheet.js):
```
/* claims-to-sheet.js — Escribe claims.json + datos regionales + config a la pestana "Datos" del Google Sheet.
   Sin dependencias npm. Usa modulos nativos https y crypto de Node.js.

   Requiere:
     - GOOGLE_SERVICE_ACCOUNT_KEY: JSON string o ruta a archivo .json con las credenciales del service account
     - config.json -> sheet.spreadsheetId

   Uso:
     node scripts/claims-to-sheet.js           # Escribe al Sheet
     node scripts/claims-to-sheet.js --dry-run  # Muestra datos sin escribir
*/
```

**Modules**: `var https = require('https'); var fs = require('fs'); var path = require('path'); var crypto = require('crypto');`

**Config loading** (same pattern as sync-from-sheet.js):
- Read config.json from ROOT
- Extract SPREADSHEET_ID from `config.sheet.spreadsheetId`
- Extract DATOS_TAB from `config.sheet.datosTab` (default "Datos")
- Parse --dry-run from process.argv

**JWT Auth function** `getAccessToken(serviceAccountKey, callback)`:
- serviceAccountKey is the parsed JSON object with `client_email`, `private_key`, `token_uri`
- Create JWT header: `{"alg":"RS256","typ":"JWT"}`
- Create JWT claim: `{"iss": client_email, "scope": "https://www.googleapis.com/auth/spreadsheets", "aud": "https://oauth2.googleapis.com/token", "iat": now_seconds, "exp": now_seconds + 3600}`
- Base64url encode header and claim
- Sign with RS256 using `crypto.createSign('RSA-SHA256')` and the `private_key` from the service account JSON
- POST to `https://oauth2.googleapis.com/token` with `grant_type=urn:ietf:params:oauth:grant_type:jwt-bearer&assertion={jwt}` (Content-Type: application/x-www-form-urlencoded)
- Parse response for `access_token`
- Call `callback(null, access_token)` or `callback(error)`

**Helper** `base64url(str)`: Convert string to base64, replace +/-/= for URL-safe encoding.

**Helper** `hashRespuesta(text)`: Return `crypto.createHash('sha256').update(text || '', 'utf8').digest('hex')` — this is the SHEET-04 hash for change detection. This is DIFFERENT from the claim's existing hash_sha256 (which hashes the extracto_verbatim). This new hash is specifically for detecting when a human edits the "respuesta" column in the Sheet.

**Data transformation function** `buildSheetData()`:
- Read claims.json, pages.json, calendar-config.json from data/ directory
- Build rows array with consistent column structure:

**Column layout for "Datos" tab** (row 1 = headers):
```
A: seccion        (CLAIMS | REGION | CONFIG)
B: id             (claim id, region slug, config key)
C: pregunta       (claim pregunta, empty for region/config)
D: respuesta      (claim respuesta, region field value, config value)
E: fuente_url     (claim fuente_url, empty for region/config)
F: fuente_referencia (claim source_reference, empty for region/config)
G: extracto_verbatim (claim extracto_verbatim, empty for region/config)
H: hash_respuesta (SHA256 of column D "respuesta" — for change detection per SHEET-04)
I: hash_verbatim  (claim hash_sha256 of extracto, empty for region/config)
J: last_checked   (claim last_checked, empty for region/config)
K: status         (claim status, empty for region/config)
L: campo          (empty for claims, field name for region/config rows like "inicio", "vacacionesFin", "year", etc.)
```

**Claims section** (50 rows): One row per claim from claims.claims array. Map each claim to the columns above. Column H = hashRespuesta(claim.respuesta).

**Region section** (16 rows): One row per region. For each region from pages.json:
- seccion = "REGION"
- id = region.regionSlug (e.g., "metropolitana")
- pregunta = "" (empty)
- respuesta = JSON.stringify of the region object (all fields in one cell for auditability)
- hash_respuesta = hashRespuesta(respuesta)
- campo = "datos_region"

**Config section** (rows for each config key + feriados): For calendar-config.json:
- One row per top-level key (year, schoolStart, winterStart, winterEnd, schoolEnd):
  - seccion = "CONFIG", id = key name, respuesta = value, campo = key name
- One row for feriados array:
  - seccion = "CONFIG", id = "feriados", respuesta = JSON.stringify(feriados), campo = "feriados"
- One row for feriadosCompletos array:
  - seccion = "CONFIG", id = "feriadosCompletos", respuesta = JSON.stringify(feriadosCompletos), campo = "feriadosCompletos"

Return `{ headers: [...], rows: [...] }` where headers is the column names array and rows is array of arrays.

**Sheet write function** `writeToSheet(accessToken, sheetData, callback)`:
- First, create/clear the "Datos" tab using `POST https://sheets.googleapis.com/v4/spreadsheets/{id}:batchUpdate` with requests:
  - Try to add a new sheet with title "Datos". If it already exists (error 400), clear it instead using `values/clear` endpoint.
- Then write all data using `PUT https://sheets.googleapis.com/v4/spreadsheets/{id}/values/{DATOS_TAB}!A1:L{totalRows}?valueInputOption=RAW` with the full data matrix (headers + rows).
- Use native https module for both calls, same pattern as sync-from-sheet.js's fetchSheet but with POST/PUT methods and Authorization: Bearer header.

**Main flow:**
1. Call `buildSheetData()` to prepare the data
2. Print summary: "X claims + Y regiones + Z config rows = total rows"
3. If --dry-run:
   - Print headers
   - Print first 5 rows as sample
   - Print "REGION" section summary (count)
   - Print "CONFIG" section summary (count)
   - Print "Dry run completo. No se escribio al Sheet."
   - Exit 0
4. If NOT --dry-run:
   - Load service account key from GOOGLE_SERVICE_ACCOUNT_KEY env var (try JSON.parse first; if fails, treat as file path and read file)
   - If no key found, error with clear message
   - Call getAccessToken -> writeToSheet -> print success
   - Exit 0 on success, 1 on error

**Error handling**: Same style as sync-from-sheet.js — console.error with descriptive messages, process.exit(1).

**IMPORTANT constraints**:
- Use `var` not `let`/`const` (project convention)
- Use `require()` not `import` (no ES modules)
- No npm dependencies — only native Node.js modules (https, fs, path, crypto)
- Callback-style async, not Promises
- Follow sync-from-sheet.js patterns for https requests (data accumulation, JSON parsing, error handling, timeout)
