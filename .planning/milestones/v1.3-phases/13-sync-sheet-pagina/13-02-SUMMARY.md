# Phase 13 Plan 02: Dynamic Tooltips from Claims.json — Summary

**One-liner:** Eliminates tech debt INT-01 by generating CLAIMS_DATA JS from claims.json and replacing 7 hardcoded BCN tooltip texts with data-claim-id driven rendering via claims-tooltips.js.

## Status

Task 1 complete. Awaiting human verification (Task 2 checkpoint).

## What Was Built

The full claims-to-frontend pipeline now works end-to-end:

1. **`scripts/generate-pages.js`** — Added two new generation blocks:
   - Reads `data/claims.json` and builds a flat `claimsDataMap` keyed by claim.id
   - Writes `public/js/claims-data.js` with `window.CLAIMS_DATA` global (50 claims)
   - Writes `public/js/claims-tooltips.js` — IIFE that runs on DOMContentLoaded, queries `[data-claim-id]` elements, and populates `.bcn-tooltip` text from CLAIMS_DATA

2. **`scripts/generate-verificacion.js`** — Updated to prefer `claims.json` over `afirmaciones.json`:
   - Tries `data/claims.json` first (v2 unified model)
   - Falls back to `data/afirmaciones.json` if claims.json missing
   - Variable aliased as `afirmaciones` for backward compatibility (same .claims + .sources structure)

3. **`public/index.html`** — Replaced 7 hardcoded tooltip spans with data-attribute pattern:
   - All 7 `bcn-badge-wrap` spans now have `data-claim-id="feriado-*"` attributes
   - All `.bcn-tooltip` spans are now empty in HTML source
   - Added `<script defer src="/js/claims-data.js">` and `<script defer src="/js/claims-tooltips.js">` before verificacion.js
