  <action>
Create scripts/notify-telegram.js as a standalone Node.js CLI script. Conventions: var (not const/let), require (not import), IIFE pattern, zero npm dependencies.

**Input contract:** Reads JSON from stdin. The JSON structure is:
```json
{
  "changes": [
    {
      "data_key": "feriado_ano_nuevo",
      "texto_antes": "old verbatim text",
      "texto_despues": "new verbatim text",
      "evaluacion": "sin_impacto|requiere_revision|actualizar",
      "claim_text": "1 de enero es feriado legal"
    }
  ],
  "total_claims_checked": 15,
  "detection_date": "2026-03-25T06:00:00Z"
}
```

**Environment variables:**
- TELEGRAM_BOT_TOKEN (required unless --dry-run)
- TELEGRAM_CHAT_ID (required unless --dry-run)

**CLI flags:**
- --dry-run: Print the formatted message to stdout instead of sending. Exit 0 without requiring env vars.

**Message format (HTML parse_mode):**
Build an HTML-formatted Telegram message. Telegram supports: `<b>`, `<i>`, `<code>`, `<pre>`, `<a href="">`. NO markdown.

Message template:
```
<b>Cambio detectado en articulado BCN</b>
<b>Fecha:</b> {detection_date}
<b>Claims revisados:</b> {total_claims_checked}
<b>Cambios:</b> {changes.length}

---

<b>Claim:</b> <code>{data_key}</code>
<b>Afirmacion:</b> {claim_text}
<b>Evaluacion IA:</b> {evaluacion_emoji} {evaluacion}

<b>Texto anterior:</b>
<pre>{texto_antes (truncated to 300 chars)}</pre>

<b>Texto actual (BCN):</b>
<pre>{texto_despues (truncated to 300 chars)}</pre>

---
(repeat for each change)

<a href="https://docs.google.com/spreadsheets/d/160WyrLOm6nV2MAg1cusYvSbVzOWnqYWIt8O5MgXRvF4">Abrir Google Sheet</a>
```

Evaluation emojis: sin_impacto = green circle, requiere_revision = yellow circle, actualizar = red circle.

**Telegram API call:**
POST to `https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage`
Body: `{ chat_id: TELEGRAM_CHAT_ID, text: messageHtml, parse_mode: "HTML" }`
Use native `require('https')` for the request.

**Telegram message length limit:** 4096 chars. If message exceeds limit, truncate text blocks further or split into multiple messages (send summary first, then details per change).

**Error handling:**
- Missing stdin JSON: print error, exit 1
- Missing env vars (non-dry-run): print error with instructions, exit 1
- Telegram API error: print HTTP status + response body, exit 1
- Success: print "Telegram notification sent" + message preview, exit 0

**Script structure (IIFE):**
```javascript
#!/usr/bin/env node
(function() {
  'use strict';
  var https = require('https');
  // ... parseArgs, readStdin, formatMessage, sendTelegram, main
})();
```
