  <action>
Create `scripts/validate-extraction.js` — CommonJS, 'use strict', var — that performs deterministic validation on visual extraction output.

**CLI interface:**
- `node scripts/validate-extraction.js` — validate data/visual-extraction.json (default)
- `node scripts/validate-extraction.js --input=PATH` — validate specific file
- `node scripts/validate-extraction.js --region=aysen` — validate single region
- `node scripts/validate-extraction.js --strict` — treat warnings as errors
- Exit code 0 = all checks pass, 1 = errors found

**Check 1 — Date format (per milestone):**
- Every `date` field matches `YYYY-MM-DD` regex: `/^\d{4}-\d{2}-\d{2}$/`
- Every `date_start` and `date_end` field matches same regex
- Date must be parseable by `new Date(value)` and not be `Invalid Date`
- Year must be 2026 (or configurable via `--year=NNNN`)

**Check 2 — Day of week (per milestone):**
- If `day_of_week` field exists, verify it matches the actual day for that `date`
- Use Spanish day names: `var DIAS = ['Domingo','Lunes','Martes','Miércoles','Jueves','Viernes','Sábado'];`
- Must handle timezone correctly: parse date as UTC to avoid off-by-one

**Check 3 — Chronological ordering (per region, per regime):**
- Within each `semestral` array, dates must be non-decreasing
- Within each `trimestral` array, dates must be non-decreasing
- `date_start` must be before `date_end` for range milestones

**Check 4 — Required milestones (per region):**
- Each region must have at least one entry with a label containing "Inicio" (school start)
- Each region must have at least one entry with a label containing "Receso de invierno" or "vacaciones" (winter break)
- Each region must have at least one entry with a label containing "Último día" or "último día" (school end)

**Check 5 — Cross-region analysis (PIPE-05):**
Define expected groups with their date signatures:
```javascript
var EXPECTED_GROUPS = {
  'ESTANDAR': {
    regions: ['antofagasta','atacama','coquimbo','valparaiso','metropolitana','ohiggins','maule','nuble','biobio','araucania','los-rios'],
    vacacionesInicio: '2026-06-22',
    vacacionesFin: '2026-07-03',
    finAno: '2026-12-04'
  },
  'NORTE': {
    regions: ['arica-y-parinacota','tarapaca'],
    vacacionesInicio: '2026-07-13',
    vacacionesFin: '2026-07-24',
    finAno: '2026-12-04'
  },
  'SUR': {
    regions: ['aysen','magallanes'],
    vacacionesInicio: '2026-06-29',
    vacacionesFin: '2026-07-17',
    finAno: '2026-12-11'
  },
  'SUR-PARCIAL': {
    regions: ['los-lagos'],
    vacacionesInicio: '2026-07-06',
    vacacionesFin: '2026-07-17',
    finAno: '2026-12-04'
  }
};
```
For each region:
- Find its expected group
- Extract receso invierno dates and fin ano date from the semestral array
- Compare against expected group values
- If mismatch: check if it matches ANY other group (= possible misclassification, warning)
- If matches no group: ERROR — likely extraction error

**Output report** (`data/extraction-validation-report.json`):
```json
{
  "timestamp": "2026-03-24T...",
  "input": "data/visual-extraction.json",
  "summary": {
    "regions_checked": 16,
    "regions_passed": 16,
    "regions_failed": 0,
    "total_errors": 0,
    "total_warnings": 0
  },
  "checks": {
    "date_format": { "passed": true, "errors": [] },
    "day_of_week": { "passed": true, "errors": [] },
    "chronological": { "passed": true, "errors": [] },
    "required_milestones": { "passed": true, "errors": [] },
    "cross_region": {
      "passed": true,
      "groups_detected": {
        "ESTANDAR": ["antofagasta", "atacama", ...],
        "NORTE": ["arica-y-parinacota", "tarapaca"],
        "SUR": ["aysen", "magallanes"],
        "SUR-PARCIAL": ["los-lagos"]
      },
      "anomalies": []
    }
  },
  "per_region": {
    "aysen": { "passed": true, "errors": [], "warnings": [], "group": "SUR" },
    ...
  }
}
```

**Console output:**
- Print each check with PASS/FAIL status
- Print per-region summary (green checkmark or red X with errors)
- Print cross-region group summary table
- Final line: `VALIDATION: PASSED (0 errors, 0 warnings)` or `VALIDATION: FAILED (N errors, M warnings)`

Use ANSI color codes sparingly (green for pass, red for fail, yellow for warning). Check `process.stdout.isTTY` before using colors.
