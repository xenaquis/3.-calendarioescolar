**3. public/js/app.js — dos cambios:**

**3a. Hacer selectRegion tolerante a bars=null:**

En la funcion selectRegion (linea 63), el loop que desactiva barras fallaria con bars=null.
Cambiar la condicion del loop de:
```javascript
for (var i = 0; i < bars.length; i++) {
```
a:
```javascript
for (var i = 0; bars && i < bars.length; i++) {
```

Esto permite llamar selectRegion(slug, null) desde el select mobile sin errores.

**3b. Agregar listener para el select en initMapSelector:**

Al FINAL de initMapSelector (antes del cierre de la funcion), agregar:

```javascript
// Wiring mobile select
var mobileSelect = document.getElementById('region-select');
if (mobileSelect) {
  mobileSelect.addEventListener('change', function () {
    var slug = mobileSelect.value;
    if (slug) {
      selectRegion(slug, null);
    }
  });
}
```

NO usar `var bars = ...` de nuevo — el NodeList ya existe en el scope superior de initMapSelector. El select pasa null como bars porque en mobile no hay barras activas que gestionar (el select mismo actua como indicador de estado activo).
