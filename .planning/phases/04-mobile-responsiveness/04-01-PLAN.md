# Plan 04-01: Mobile Responsiveness & CSS Token Fixes

## Objective

Fix missing CSS tokens (`--space-5`, `--leading-relaxed`) that are referenced but undefined,
and improve mobile responsiveness of key-facts cards so dates render correctly on small screens.

## Tasks

### Task 1: Fix missing CSS tokens (type=auto)

Add `--space-5: 1.25rem` and `--leading-relaxed: 1.75` to tokens.css.
These are referenced in components.css but missing from tokens.css, causing browsers to
fall back to `unset` (effectively 0 for spacing, inherited for line-height).

Files: `public/css/tokens.css`

### Task 2: Improve key-fact date font responsiveness (type=auto)

On mobile (< 380px), `key-fact__date` at `var(--text-2xl)` (1.5rem) can overflow.
Add a mobile media query to reduce font size to `var(--text-xl)` (1.25rem)
for screens under 400px.

Files: `public/css/components.css`

### Task 3: Update BLUEPRINT.md (type=auto)

Record the CSS fixes in BLUEPRINT.md.

Files: `BLUEPRINT.md`

## Success Criteria

- `--space-5` defined in tokens.css
- `--leading-relaxed` defined in tokens.css
- key-fact__date reduces size on very small screens
- BLUEPRINT.md updated
